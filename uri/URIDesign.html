Designing URIs

Basic Construction of a URI

The following identifies the elements within a URI and forms the basic pattern:

http://{domain}/{concept}/{type} /{reference}

  • {Domain}: In the case of this guide, it is recommended that the publisher use 'data.govt.nz'. The sub-domain will need to be determined carefully.
  • {concept}: A word or string to capture the essence of the real-world 'Thing' that the set names. For example, environment. A concept MUST be: lowercase, words separated by hyphens and singular (not 'environments').
  • {type}: Refer to the next section 'URI Types' below.
  • {reference}: A speciifc item, term or concept.

Types of URIs

The following table describes each type of URI with associated resource: Also refer to Australia Guidelines...

Type of Resource Type of URI Definition
A real-world 'Thing' (Non-Information Resource) Identifier URI Identifies some physical or abstract real-world 'Things'. Example of a real-world 'Thing': a person, a road, a lake. Example of abstract real-world 'Thing': a government, a council.
---
A regular document (Information Resource) Document URI Identifies a document. The Document URI should resolve directly to a document on the Web.
---
Definitions of characteristics of a 'Real-World Thing' (Information Resource> Ontology URI Definitions of concepts and relations contained within the ontologies that define characteristics of a real- world 'Thing'. Example: a person class with properties such as 'firstName', 'lastName' etc. The ontology URI with return a definition or set of definitions about the real-world 'Thing'. That is the person class will return the properties listed above.

10 Rules for persistent URIs

Based on a survey across EU agencies and services, EU Member States and standardisation bodies where management of URIs have been subject to policy. The best practices can be viewed as a diagram and summarised as: Read full report.

    DO
  • 1: Follow the pattern -- http://{domain}/{concept}/{type}/{reference}
  • 2: Re-use existing identifiers -- http://data.govt.nz/environment/doc/1234
  • 3: Link multiple representations -- http://data.govt.nz/environment/doc/1234.rdf | http://data.govt.nz/environment/doc/1234.html
  • 4: Implement 303 redirects for real-world objects
  • 5: Use a dedicated service i.e., independent of data orginator
    DON'T
  • 6: Avoid stating ownership -- http://data.govt.nz/environment/doc/HorizonsDC/
  • 7: Avoid version numbers -- http://data.govt.nz/environment/doc/erosion/v0.1/1234
  • 8: Avoid using auto-increment -- http://data.govt.nz/environment/doc/erosion/1234 & http://data.govt.nz/doc/environment/1235
  • 9: Avoid query strings -- http://data.govt.nz/environment/doc/erosion?id=123456
  • 10: Avoid file extensions --http://data.govt.nz/environment/doc/erosion/12345.csv
