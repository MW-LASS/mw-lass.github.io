URI Principles

W3C Linked Data Principles

URIs underpin linked data, so without URIs, users would not have the capability of following links to other related resouces on the Web. Tim Berners-Lee identified four main principles when applying linked data: Read more...

  • 1. Use URIs as names for things
  • 2. Use HTTP URIs so that people can look up those names.
  • 3. When someone looks up a URI, provide useful information, using the standards (RDF*, SPARQL).
  • 4. Include links to other URIs. so that they can discover more things.

General Design Principles

The following table identifies additional specific design principles for publishing linked data based on experience gained from the Australian Government's Guide for URIs.
10 Provide a human-readable representation in HTML at the Dataset URI. MUST
11 If multiple representations exist, provide a means of discovering specific URIs for each of the available representations. SHOULD
12 The license for inspection or use of the Linked Dataset shall be provided using a common vocabularly and using NZGOAL. SHOULD
13 The metadata for a Linked Dataset should be provided using a common vocabulary and contain the expected longevity and maintenance plans for the Dataset URI. SHOULD
14 The current technical implementation of a data publication system should not be visible in or otherwise affect the URI for a Linked Dataset. SHOULD NOT
